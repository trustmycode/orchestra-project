## Описание
Реализовать агента, который анализирует два шага и предлагает схему передачи данных (плейсхолдеры).

## Ключевые шаги
1.  Добавить инструменты для получения схем запроса/ответа шагов.
2.  Агент анализирует семантическую близость полей (например, `id` в ответе и `orderId` в запросе) и генерирует JSON с рекомендацией маппинга.

## Критерии приемки
-   UI конструктора предлагает пользователю готовые плейсхолдеры для связи шагов.
