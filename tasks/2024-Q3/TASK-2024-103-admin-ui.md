## Описание
Разработать в `orchestra-web` отдельный раздел "Администрирование", доступный только пользователям с ролью `ORG_ADMIN` или `SUPER_ADMIN`.

## Ключевые шаги
1.  Создать новый роут и базовый layout для админ-панели.
2.  Перенести в этот раздел UI для управления лимитами (`TASK-2024-088`).
3.  Добавить UI для управления пользователями и их ролями в рамках тенанта.
4.  (Опционально) Добавить просмотрщик `audit_logs`.
5.  (Опционально) Добавить UI для запуска онбординга нового тенанта (`TASK-2024-102`).

## Критерии приемки
-   Пользователи с соответствующими правами видят и могут использовать раздел "Администрирование".
-   В разделе доступен как минимум функционал по управлению лимитами `Scheduler`'а.
