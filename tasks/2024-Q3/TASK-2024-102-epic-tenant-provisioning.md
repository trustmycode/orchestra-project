## Описание
Создать автоматизированный или полуавтоматизированный процесс для создания нового тенанта в системе. Это включает создание всех необходимых ресурсов: запись в БД, конфигурация в Keycloak, создание очередей в RabbitMQ.

## Ключевые шаги
1.  Создать `ProvisioningService` в `orchestra-api`.
2.  Реализовать в нем логику, которая через API соответствующих систем (Keycloak Admin API, RabbitMQ Management API) создает и настраивает ресурсы для нового тенанта.
3.  Создать для этого сервиса защищенный API-эндпоинт или CLI-команду.

## Критерии приемки
-   Выполнение одной команды/запроса приводит к полному созданию и настройке нового тенанта во всех компонентах системы.
