## Описание

Реализовать возможность добавления, редактирования и использования секретов (паролей, токенов) через UI без перезагрузки сервисов. Секреты должны храниться в БД и подставляться в параметры подключения (`DbConnectionProfile`, `KafkaClusterProfile`) непосредственно перед выполнением теста.

## Критерии приемки

- Пользователь может создать секрет `MY_PASS` в UI.
- Пользователь может использовать `{{secret.MY_PASS}}` в поле пароля БД.
- При запуске теста `Executor` успешно подключается к БД, используя значение секрета.
- Секреты изолированы между тенантами.
