## Описание
Создать интерфейсы для управления аннотациями и правилами `Data Resolver`, чтобы администраторы могли явно связывать поля API с сущностями и критериями.

## Ключевые шаги
1. Добавить страницу/раздел «Data Resolver Registry», где перечислены поля запросов и связанные сущности.
2. Позволить выбирать резолверы и задавать дефолтные критерии через UI.
3. Сохранить конфигурацию и отображать статус синхронизации/последнего обновления.

## Критерии приемки
- Пользователь может настроить связь `requestField → entity/resolver` без редактирования конфигов вручную.
- Эти настройки учитываются при генерации `DataPlan` и исполнении `Data Resolver`.
