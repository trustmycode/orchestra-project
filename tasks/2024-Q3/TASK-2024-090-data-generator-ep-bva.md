## Описание
Модифицировать `ai-service` (и/или `Data Resolver`), чтобы он мог генерировать несколько наборов данных для одного пути на основе техник Equivalence Partitioning (EP) и Boundary Value Analysis (BVA).

## Ключевые шаги
1.  Создать детерминированный компонент, который по `Path Predicate` (например, `income > 50000`) вычисляет граничные и эквивалентные значения (`50000`, `50001`, `60000` и т.д.).
2.  Обновить `ai-service`, чтобы он принимал эти вычисленные значения и "оборачивал" их в полноценный JSON, дозаполняя остальные поля.
3.  Реализовать логику создания нескольких `TestDataSet` для одного `TestScenario`, покрывающих разные классы данных.

## Критерии приемки
-   Для сценария с условием `income > 50000` автоматически создается несколько наборов данных, включая `income=50001` (граница) и `income=49999` (негативный).
